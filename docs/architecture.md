# iPAS AIAP Learning OS x CM 架構說明

## 1. 核心概念

Learning OS 不是單純的網頁，也不是單純的題庫。

它比較像一個「教學作業系統」，負責把課程、題目、新聞、新知、審核流程串起來。

CM 是 Content Mother，也就是內容母體。它保存課程內容、題庫、新聞素材、官方文件與 AI 輔助產出的草稿，並讓老師與助教可以控管品質。

## 2. 架構分層

```text
學生使用層
  Local HTML：離線學習、練題、查看進度

資料交換層
  JSON：題庫、新聞、課程模組、審核狀態

內容母體層
  GAS + Google Sheet：保存正式內容與版本

知識來源層
  新聞、官方文件、課程講義、NotebookLM 摘要

品質控管層
  AI 輔助產題 + 助教整理 + 老師審核
```

## 3. 為什麼採用 Local-first

Local-first 的意思是：學生端不依賴網路也能使用。

這樣有三個好處：

- 課堂展示穩定，不怕網路或登入問題
- 學生可以把檔案帶回家自學
- 系統可以先教概念，再逐步接上雲端同步

雲端不是消失，而是放在 CM 端負責內容管理與更新。

## 4. 模組邊界

| 模組 | 負責什麼 | 不負責什麼 |
| --- | --- | --- |
| Local HTML | 學習體驗、練題、匯入匯出 | 長期保存正式資料 |
| JSON 題庫 | 統一資料格式 | 決定題目品質 |
| GAS CM | 雲端保存、版本更新、資料發布 | 取代老師審核 |
| 新聞轉題 | 把新知轉成考點草稿 | 自動成為正式考題 |
| 人工審核 | 檢查正確性、難度、考科對應 | 大量重複產生草稿 |

## 5. 典型流程

### 學生自學流程

1. 開啟本地 HTML
2. 選擇課程模組
3. 閱讀核心概念
4. 練習題目
5. 查看解析與標籤
6. 需要時匯入老師更新的題庫 JSON

### 助教內容整理流程

1. 收集 AI 新聞、官方文件或課堂補充
2. 摘出考點
3. 用 AI 產生題目草稿
4. 標記章節、難度、來源與審核狀態
5. 交給老師審核

### 老師控管流程

1. 查看草稿題
2. 修改題幹、選項與解析
3. 判斷是否符合 iPAS 考科方向
4. 核准後發布為正式題
5. 匯出新版 JSON 給學生端使用

## 6. 新聞轉題原則

新聞不能直接變成考題。

正確流程是：

```text
新聞事件
→ 抽象化成 AI 概念
→ 對應 iPAS 考科能力
→ 形成學習目標
→ 產生題目草稿
→ 人工審核
→ 正式題庫
```

例如：

```text
某公司推出新的生成式 AI 功能
→ 生成式 AI 可以改變工作流程
→ 對應「AI 應用情境規劃」
→ 學生需能判斷導入效益與風險
→ 產生情境題
→ 老師審核
```

## 7. MVP 與正式版差異

| 項目 | MVP | 正式版 |
| --- | --- | --- |
| 學生端 | 單一 HTML | 可版本更新的離線包 |
| 題庫 | 內嵌示範資料 | GAS/Sheet 發布 JSON |
| 新聞轉題 | 離線模擬 | 串接新聞來源與 AI 摘要 |
| 審核 | 本地狀態 | 老師後台與審題紀錄 |
| 統計 | localStorage | 班級與學生資料庫 |
