## 任務目標

請根據 2026/04/10 最新簡章，建立一份可用於題庫系統的「考科分類資料庫」。

資料庫需要支援兩件事：

1. 保存官方簡章的初階 / 中階考科分類，整理到最細項。
2. 將目前兩次初階 A1 / A2 題目，對應到官方分類項目。

題目 mapping 不一定要到最細的第三層。建議策略是：

- 優先對應到第二層代碼，例如 `L111`、`L123`
- 若題目很明確，也可以補第三層，例如 `L11101`
- 若題意跨多個主題或無法穩定判斷，可只對應第一層，例如 `L11`
- 每筆 mapping 都要保留信心分數與簡短理由

## 建議資料表二：question_topic_mapping

用途：保存題目與官方分類的對應。

建議欄位：

| 欄位 | 說明 |
| --- | --- |
| `question_id` | 原題庫 `編號`，例如 A1-001 |
| `subject_code` | A1、A2 |
| `exam_year` | 例如 2025、2026 |
| `exam_session` | 例如 第4梯次、第1次 |
| `question_text` | 題目 |
| `option_text` | 可合併四個選項，供判斷用 |
| `mapped_l1_code` | 對應第一層代碼 |
| `mapped_l2_code` | 對應第二層代碼，優先填 |
| `mapped_l3_code` | 可選，只有很明確才填 |
| `mapping_depth` | `L1` / `L2` / `L3` |
| `confidence` | 0.00 到 1.00 |
| `mapping_reason` | 一句話說明判斷依據 |
| `keywords` | 用逗號列出判斷關鍵詞 |
| `needs_review` | TRUE / FALSE |

## Mapping 原則

1. A1 題目只能對應到 `L11` 底下的分類。
2. A2 題目只能對應到 `L12` 底下的分類。
3. 預設以第二層 `Lxxx` 為主要 mapping 粒度。
4. 第三層 `Lxxxxx` 只有在題目非常明確時才填，避免過度猜測。
5. 若一題同時涵蓋多個觀念，選擇最主要考點；必要時可另外加 `secondary_l2_code`。
6. 若題意模糊，填第一層並將 `needs_review` 設為 TRUE。
7. 每筆都要附上 `mapping_reason`，方便人工複查與教學說明。
